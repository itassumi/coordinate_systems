# Дослідження методу Градієнтного Спуску для мультилатерації

## Завдання 1: Контрольний запуск ("Ідеальний світ")

### Параметри:
- NOISE_LEVEL = 0
- LEARNING_RATE = 0.01
- INITIAL_GUESS = [50000, 50000]
- TRUE_POSITION = [45000, 35000]

### Результати:
- Фінальна похибка: 0.00 метрів
- Знайдена позиція: (45000.00, 35000.00)

### Графіки:
![Завдання 1](https://github.com/itassumi/coordinate_systems/blob/7e5a9bf29b99636493cfbbf39505f43bcda51d17/3/1.png)

### Відповіді на питання:
1. **Чому похибка не дорівнює 0.00 метрів?** 
   У цьому ідеальному випадку похибка дорівнює 0.00 метрів, що демонструє ідеальну роботу алгоритму за відсутності шуму.

2. **Як поведінка градієнта пов'язана з кривою навчання?**
   Градієнт швидко зменшується, що відповідає стрімкому падінню функції втрат, що свідчить про ефективну збіжність алгоритму.

## Загальний висновок

Найбільший вплив на точність позиціонування мають:
1. **Рівень шуму** у вхідних данах - прямий вплив на якість результату
2. **Геометрія розташування** базових станцій (GDOP) - погана геометрія значно посилює вплив шуму
3. **Швидкість навчання** алгоритму - невідповідний вибір може призвести до розбіжності або стагнації

На швидкість збіжності впливають:
1. **Початкове припущення** - ближче до справжньої позиції = швидша збіжність
2. **Швидкість навчання** - оптимальне значення забезпечує швидку і стабільну збіжність
3. **Параметр TOLERANCE** - визначає коли алгоритм зупиняється

Для реальних систем необхідно знаходити баланс між точністю та обчислювальними витратами, враховуючи обмеження, накладені якістю даних та геометрією системи. Найкращі результати досягаються при комбінації хороших вхідних даних, правильної геометрії системи та оптимальних параметрів оптимізації.
