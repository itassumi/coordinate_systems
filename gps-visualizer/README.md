# Лабораторна робота

## **Розробка додатку для візуалізації вимірювань GPS**

## Опис роботи

Розробка веб-додатку для візуалізації GPS-вимірювань у реальному часі.
Додаток підключається до емулятора GPS, отримує дані про положення супутників та розраховує координати об'єкта двома методами: аналітичним та чисельним.

## Виконані завдання

### 1. Налаштування емулятора GPS

* Запуск Docker контейнера з емулятором вимірювальної частини GPS
* Налаштування WebSocket-з'єднання на порті 4001
* Конфігурування параметрів емуляції через REST API

### 2. Розробка веб-інтерфейсу

* Створення HTML-структури з трьома основними секціями
* Реалізація форми керування параметрами емулятора
* Налаштування Canvas для візуалізації даних
* Стилізація інтерфейсу за допомогою CSS

### 3. Реалізація алгоритмів розрахунку

* **Аналітичний метод** — точне розв'язання системи рівнянь
* **Чисельний метод** — мінімізація функції похибки (numeric.js)
* Обробка даних у реальному часі через WebSocket

### 4. Візуалізація результатів

* Відображення супутників (сині точки)
* Відображення позиції об'єкта аналітичним методом (зелена точка)
* Відображення позиції об'єкта чисельним методом (червона точка)
* Координатна сітка для кращої орієнтації

## Технічні деталі реалізації

### Використані технології

* **Frontend:** HTML5, CSS3, JavaScript (ES6+)
* **Візуалізація:** Canvas API
* **Математичні обчислення:** numeric.js
* **Комунікація:** WebSocket, REST API
* **Емуляція:** Docker

## Результати роботи додатку

### Підключення до емулятора

```
WebSocket статус: Підключено до емулятора GPS
Порт: 4001
Протокол: WebSocket
```

### Приклад отриманих даних

```json
{
  "id": "affd1x0f7-d6b0-ddfc-9751-545f8f78bd0",
  "x": 98.98877479575082,
  "y": 77.2531956554876,
  "sentAt": 176436393489,
  "receivedAt": 1764363934839.224
}
```

### Порівняння методів розрахунку

| Метод           | Принцип роботи                    | Переваги                     | Недоліки                         |
| --------------- | --------------------------------- | ---------------------------- | -------------------------------- |
| **Аналітичний** | Точне розв'язання системи рівнянь | Точність для ідеальних даних | Чутливість до похибок вимірювань |
| **Чисельний**   | Мінімізація сумарної похибки      | Стійкість до похибок         | Наближений результат             |

## Аналіз результатів

### Коректність роботи

* WebSocket підключення працює стабільно
* Дані надходять у реальному часі
* Обидва методи розрахунку повертають коректні координати
* Візуалізація працює правильно

### Продуктивність

* Оновлення даних відбувається раз на секунду
* Чисельний метод потребує більше ресурсів
* Аналітичний метод швидший, але менш стійкий до шумів

## Висновки

1. WebSocket — ефективний спосіб передавання даних у реальному часі
2. Аналітичний метод підходить для ідеальних даних без шуму
3. Чисельний метод краще працює за реальних умов
4. Canvas API забезпечує плавну та наочну візуалізацію

## Висновок

Під час виконання лабораторної роботи було створено веб-додаток для візуалізації GPS-вимірювань у реальному часі. Реалізовано два методи визначення координат, проведено їх порівняння та підтверджено, що чисельні методи є стійкішими до похибок, що робить їх більш придатними для реальних GPS-сценаріїв.
